This blog sits beside the interactive [Sudip AI Playbook](https://sudip-ai-playbook.github.io/) — short notes on architecture choices, consulting practice, and how the tools fit together.

## What you will find here

- Cross-cloud AI architecture patterns (AWS, Azure, Google Cloud)
- ConsultAI OS lifecycle notes: stages, workshops, gates, and deliverables
- LLM FinOps: cost models, alternatives, and commercial trade-offs
- Release notes and how-to guides for the playbook itself

## How this relates to the app

The interactive playbook remains at the site root. Use it to compare services, score decisions, run workshops, and export briefs. Use this blog to capture the **why** behind those moves — for stakeholders, future-you, and the team.

## Write your next post

Add a Markdown/MDX file under `website/blog/` with a front matter block (`title`, `authors`, `tags`), then open a PR. Local preview:

```bash
cd website
npm start
```

The blog deploys to `/blog/` on every push to `main`.
