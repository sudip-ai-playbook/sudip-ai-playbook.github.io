# Core deliverables you should master

<ExecSummary>
These artefacts are the practical proof of the Learning Map. If you can produce them to VALUE standard, you can operate as an AI Solution Engineer on live work.
</ExecSummary>

## Qualification

- Opportunity brief
- Stakeholder map
- Qualification scorecard
- Initial value hypothesis
- Go/no-go recommendation

## Discovery

- Discovery agenda and interview questions
- Process map
- Problem statement
- Current-state assessment
- Requirements catalogue

## Assessment

- Data-readiness assessment
- AI feasibility assessment
- Security and governance assessments
- Build-buy-partner assessment
- Use-case prioritisation

## Design

- Target operating model
- Architecture and data-flow diagrams
- Threat model
- Integration design
- Evaluation strategy
- Responsible AI controls

## Prototype

- Prototype backlog and experiment plan
- Evaluation dataset
- Demonstration script
- Findings report
- Production-readiness gap analysis

## Commercial proposal

- Executive summary, scope, deliverables, assumptions, dependencies
- Timeline, team, cost, benefits, risks, acceptance criteria

## Implementation

- Delivery roadmap and RAID log
- Architecture decisions
- Test strategy and deployment plan
- Change-management plan and governance evidence

## Transition and operation

- Runbook and support model
- Monitoring dashboards and incident process
- Knowledge-transfer plan
- Benefits tracking and optimisation roadmap

<NextSteps>

Use these as a checklist against the [Competency test](./competency-test). Practice producing one artefact per week from a live or fictional case.

</NextSteps>
