# How to use this Learning Map

<ExecSummary>
Focus on **how you will use each concept in practice**. For every topic: read the summary, study the Learn list, work the real-world scenario, complete the practice exercise, and file the output. Expand thin sections over time with your own industry examples.
</ExecSummary>

## Page template (every topic)

Each topic page follows the same structure:

1. **Executive summary** — why it matters
2. **When to use** — when to study it
3. **Learn** — concepts and frameworks
4. **Real-world scenario** — a concrete enterprise example
5. **Practice exercise** — something you write or decide
6. **Questions you must answer** — interview and client-ready
7. **Expected outputs** — artefacts to master
8. **Checklist + next steps**

## How to expand a topic in depth

When you want more depth on a bullet (for example “NPV” or “GraphRAG”):

1. Add a subsection under **Learn** with definition → when to use → pitfalls.
2. Add a second **Real-world scenario** in your industry.
3. Add a **worked example** (numbers, diagram, or decision table).
4. Link the related Guide chapter and an app tool (FinOps, ConsultAI, Compare).
5. Store the artefact in your personal pattern library (`35. Personal Effectiveness`).

## Weekly practice loop

| Day | Action |
| --- | --- |
| Mon | Pick one topic from your gap list |
| Tue–Wed | Complete Learn + scenario notes |
| Thu | Finish practice exercise artefact |
| Fri | Run VALUE on the artefact; file it |

## Integration with the rest of the playbook

- **Learning Map** builds capability.
- **Guide** tells you how to run an engagement step.
- **8D / VALUE** gate quality and stage advance.
- **App** is where you workshop, compare and export.

Deep-link, do not duplicate: when a Learning Map topic overlaps a Guide page, link it and keep practice exercises here.
