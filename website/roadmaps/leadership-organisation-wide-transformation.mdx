# Organisation-Wide Transformation Roadmap

A Managing Director should not only advise clients on transformation. They should ensure the firm demonstrates the same discipline, ambition and execution capability it promises to the market.

Internal transformation is a strategic leadership responsibility: redesign how the organisation creates value—not merely introduce more technology.

<ExecSummary>
Treat the firm as a major transformation client. Define a transformation thesis, run an enterprise portfolio against a few strategic outcomes, and require every major programme to have an accountable owner, baseline, target operating model, owned benefits, workforce plan, adoption strategy, delivery roadmap and transparent reporting.
</ExecSummary>

## Why executive leadership is required

Transformation fails when it crosses boundaries that no single project or function can resolve: processes, platforms, data ownership, funding, structures, skills, risk, incentives, service-line responsibilities and governance.

The Managing Director creates the conditions for success: strategic case, aligned outcomes, accountable owners, investment, conflict resolution, protection from short-term pressure, benefit focus, workforce and adoption attention, delivery challenge and personal modelling.

## Capability map

Thirteen transformation domains plus eight mandatory programme components:

| Domain | Leadership shift |
| --- | --- |
| **Enterprise portfolio** | Connect initiatives to a few strategic outcomes; stop duplicate and low-value work |
| **Transformation thesis** | Market change, internal constraints, future organisation, value creation, leadership behaviour |
| **Skills-based staffing** | Enterprise skills taxonomy, matching, visibility across service lines |
| **Workflow automation** | Burden → simplify → automate → redesign roles and controls |
| **Knowledge management** | Search, ownership, curation, reuse and IP protection—not tools alone |
| **Proposal creation** | Reuse and AI drafts free time for insight, differentiation and commercial quality |
| **Finance and HR** | Journey-based employee experience; service quality metrics |
| **Workforce planning** | Skills, activities and capabilities—not only headcount |
| **Delivery centres** | Technology-enabled scale integrated into the operating model |
| **Engagement economics** | Full lifecycle from qualification to knowledge capture |
| **Admin simplification** | New initiatives must remove more complexity than they add |
| **Common platforms** | Shared foundations; differentiate where strategy requires it |
| **Culture and resistance** | Model behaviour; diagnose resistance type before responding |

## Transformation thesis (five questions)

1. What is changing in the market?
2. What internal constraints are holding the firm back?
3. What future organisation is required?
4. How will value be created?
5. What must leaders do differently?

Example thesis:

> The firm will become a technology-enabled professional-services organisation in which common platforms, intelligent automation and skills-based workforce models allow professionals to spend more time on judgement, client relationships and complex problem-solving.

## Portfolio outcomes (limit and link)

Every initiative must contribute to one or more of:

- Client service
- Employee productivity
- Quality and risk
- Engagement profitability
- Speed to market
- Scalable delivery
- Workforce modernisation
- Technology estate simplification
- Management information
- Reusable digital assets

### Portfolio questions (regular)

| Ask | Decision implication |
| --- | --- |
| Strategic vs mandatory vs discretionary? | Funding and sequencing |
| Measurable financial or client value? | Continue or redesign |
| Duplicating other work? | Combine or stop |
| Same scarce people, data or systems? | Sequence or share |
| Accelerate, redesign, combine or stop? | Explicit portfolio action |

## Eight mandatory components (every major programme)

| # | Component | Without it |
| --- | ---: | --- |
| 1 | **Senior accountable owner** | Drift; technology delivered without business outcome |
| 2 | **Measurable baseline** | Unverifiable benefit claims |
| 3 | **Target operating model** | New tools around old ways of working |
| 4 | **Clear owned benefits** | Vague productivity promises |
| 5 | **Workforce plan** | Roles change silently; adoption stalls |
| 6 | **Adoption strategy** | Go-live mistaken for success |
| 7 | **Delivery roadmap** | No decision gates; endless continuation |
| 8 | **Transparent reporting** | Concealed problems; status theatre |

### Benefit types to own

| Type | Examples |
| --- | --- |
| **Financial** | Margin, cost, utilisation, avoided spend |
| **Operational** | Cycle time, quality, standardisation, scalability |
| **Strategic** | Client proposition, credibility, innovation, market entry |
| **Risk** | Controls, auditability, compliance, incidents |

Each benefit needs definition, baseline, target, owner, method, date and evidence.

## Governance that enables decisions

| Forum | Owns |
| --- | --- |
| **Executive transformation board** | Direction, investment, portfolio, major risk, benefits, escalation |
| **Transformation portfolio office** | Visibility, standards, dependencies, finance, assurance, reporting |
| **Design authorities** | Architecture, data, cyber, responsible AI, process, workforce, commercial |
| **Business adoption network** | Service-line and regional champions, process owners, learning, communications |

## First 90 days

| Phase | Days | Focus |
| --- | ---: | --- |
| **Diagnose** | 1–30 | Portfolio, investment, risks, adoption, duplication, baselines, urgent decisions, user conversations |
| **Align** | 31–60 | Thesis, outcomes, owners, decision rights, stop/combine, platform principles, benefits, funding, reporting |
| **Mobilise** | 61–90 | Revised portfolio, blockers, pilots, governance, change network, legacy retirement, benefit reviews |

## Outcome metrics (not activity)

Prefer:

- Professional hours released
- Engagement margin improvement
- Proposal turnaround and win quality
- Reuse and automation savings
- Process errors and risk exposure
- Employee and client experience
- Technology cost and mobilisation speed
- Actual adoption and legacy retirement

Avoid counting workshops, training completions, features shipped or pilots launched as transformation success.

## Failure modes to confront

| Failure | Correction |
| --- | --- |
| Technology treated as the transformation | Redesign process, roles and incentives |
| Unclear ownership | One accountable outcome owner with authority |
| Vague benefits | Baseline, target, owner, evidence |
| Local platform protection | Enterprise platform principles and decision rights |
| Late adoption | Design-time change; retire legacy tools |
| Avoided workforce impact | Role-impact and reskilling plan |
| Too many simultaneous initiatives | Portfolio stop/combine discipline |
| Leaders fully delegate transformation | Active sponsorship and personal modelling |
| Weak programmes never stopped | Courage to correct, reduce or stop |

## Evidence of readiness

Before claiming organisation-wide transformation leadership is in place, confirm:

- [ ] Published transformation thesis answering the five questions
- [ ] Enterprise portfolio mapped to a limited set of strategic outcomes
- [ ] Explicit accelerate / redesign / combine / stop decisions in the last review cycle
- [ ] Every major programme has the eight mandatory components
- [ ] Executive board, portfolio office, design authorities and adoption network have clear decision rights
- [ ] Benefit baselines approved before major delivery
- [ ] Adoption measures and legacy-retirement dates on live programmes
- [ ] Reporting answers outcomes, blockers and decisions—not activity theatre
- [ ] Internal programmes challenged with the same assurance methods used for clients
