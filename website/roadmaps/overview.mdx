# Engineering Roadmaps

Roadmaps in this section turn capability maps into **ordered learning and delivery journeys**: what to learn first, what to build next, and what must be true before production—or before you can lead an engineering organisation effectively.

<ExecSummary>
Use Roadmaps when you need a stage-by-stage path from problem to production, security operations or leadership—not only concepts, but the skills, controls and artefacts required to ship and scale safely.
</ExecSummary>

<WhenToUse>
- Use alongside the [Learning Map](/learning-map/overview) for practice-first topics and the [Guide](/ai-solution-engineering/overview) for engagement playbooks.
- Prefer a roadmap when the goal is to run an end-to-end AI engagement, build a production capability (for example AI engineering, agents, DevSecOps, API security, AI red teaming, enterprise AI security or Model FinOps), ship an AI product, or build a leadership operating system.
- Pair deep dives with [Articles](/articles) for long-form engineering guides.
</WhenToUse>

## What a good roadmap includes

1. **Prerequisites** — foundations required before the journey
2. **Core capability** — skills, systems and artefacts to build
3. **Production or operating discipline** — quality, delivery, cost and risk controls
4. **Trust and people systems** — security, governance, coaching and culture
5. **Evidence of readiness** — checklists, metrics, ownership and maturity levels
