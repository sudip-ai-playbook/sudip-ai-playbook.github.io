# Engineering Roadmaps

Roadmaps in this section turn capability maps into **ordered learning and delivery journeys**: what to learn first, what to build next, and what must be true before production.

<ExecSummary>
Use Roadmaps when you need a stage-by-stage path from foundations to production—not only concepts, but the engineering skills, controls and artefacts required to ship safely.
</ExecSummary>

<WhenToUse>
- Use alongside the [Learning Map](/learning-map/overview) for practice-first topics and the [Guide](/ai-solution-engineering/overview) for engagement playbooks.
- Prefer a roadmap when the goal is to build a production capability (for example agents), not only to study a single topic.
- Pair deep dives with [Articles](/articles) for long-form engineering guides.
</WhenToUse>

## What a good roadmap includes

1. **Prerequisites** — backend, APIs, Git and security foundations
2. **Core capability** — models, prompts, tools, memory and architecture
3. **Production engineering** — evaluation, observability, cost and CI/CD
4. **Trust controls** — permissions, injection defence, privacy and red-team testing
5. **Evidence of readiness** — checklists, metrics and ownership
